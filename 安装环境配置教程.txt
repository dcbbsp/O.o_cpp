C++的环境配置十分麻烦,和一些语言不同,C++在编译器写好代码后在用配置好的
编译器进行转换,转换成计算机可执行的程序.如果幸运的一次可以配置好,如果失
败了再多看几遍试试.

首先本地环境设置,需要确定电脑上有两个必需品: "文本编辑器和C++编译器"

文本编译器: 就是用来方便你写代码运行调试的软件,可以帮助你的开发效率
C++编译器: 目标都和翻译器一样,将C++源代码分析编译成计算机可以解读运行的二进制

1.文本编辑器包括:
Windows Notepad、OS Edit command、Brief、Epsilon、EMACS 和 vim/vi.(菜鸟文档)

2.文本编辑器的名称和版本在不同的操作系统上可能会有所不同。例如，Notepad 通常用于 Windows 操作系统上，vim/vi 可用于 Windows 和 Linux/UNIX 操作系统上.(菜鸟文档)

3.通过编辑器创建的文件通常称为源文件，源文件包含程序源代码.C++ 程序的源文件通常使用扩展名 .cpp、.cp 或 .c(菜鸟文档)

4.C++ 编译器
    写在源文件中的源代码是人类可读的源。它需要"编译"，转为机器语言，这样 CPU 可以按给定指令执行程序。

    C++ 编译器用于把源代码编译成最终的可执行程序。

    大多数的 C++ 编译器并不在乎源文件的扩展名，但是如果您未指定扩展名，则默认使用 .cpp。

    最常用的免费可用的编译器是 GNU 的 C/C++ 编译器，如果您使用的是 HP 或 Solaris，则可以使用各自操作系统上的编译器。

    以下部分将指导您如何在不同的操作系统上安装 GNU 的 C/C++ 编译器。这里同时提到 C/C++，主要是因为 GNU 的 gcc 编译器适合于 C 和 C++ 编程语言(菜鸟文档)
